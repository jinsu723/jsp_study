1. 쿠키와 세션
	1) 쿠키(Cookie)
		웹 서버에서 클라이언트에게 전송해주는 작은 데이터 조각이며 사용자의 웹 브라우저에 저장된다
		같은 서버에 요청을 계속 보내게 되면 웹 브라우저가 요청 헤더에 담아서 보내준다
		서버에서 관리하지 않기 때문에 서버의 부담이 줄어들지만 클라이언트에 저장되는 정보기 때문에 보안상 문제가 발생할 수 있으며 클라이언트 쪽에서 쿠키를 삭제하지 않으면 쿠키는 정해진 시간동안 계속 남아있게 된다

		클라이언트의 브라우저에서 쿠키 허용 여부를 체크하므로 허용하지 않으면 쿠키로 구현한 서비스는 무용지물이 될 때가 많다

	2) 쿠키의 동작 방식
		- 클라이언트가 서버에 요청을 보낸다
		- 웹 서버에서 쿠키를 생성하여 클라이언트에게 응답을 보낸다
		- 브라우저가 쿠키를 저장한다
		- 이 후 같은 서버로 요청을 보내게 되면 요청에 쿠키를 담아서 보낸다
		- 서버는 해당 쿠키에서 필요한 데이터를 꺼내서 사용한다

	3) 쿠기(Cookie)
		- 쿠키 생성/사용

		- 쿠키를 생성, 요청에 담기
			Cookie cookie = new Cookie("쿠키이름", "쿠키값");
			cookie.setMacAge(60 * 60 * 24); // 쿠키의 유효기간(초단위)
			// 만약 쿠키 유효기간을 설정하지 않으면 브라우저 종료 시 삭제
			response.addCookie(cookie);

		- 쿠키 유무 확인, 쿠키의 데이터 가져오기
			String cookieString = request.getHeader("Cookie");
			// 쿠키에 저장된 값을 문자열 형태로 가져옴

			Cookie[] cookie = request.getCookies();
			// 쿠키를 배열 형태로 가져옴

			if(cookies != null) {
				for(Cookie cookie : cookies) {
					cookie.getName();
					cookie.getValue();
				}
			}

		- 쿠키 삭제
			Cookie[] cookies = request.getCookies();
			if(cookies != null) {
				for(Cookie cookie : cookies) {
					cookie.setMaxAge(0);  // 유효기간을 0으로 변경하여 삭제한다
					response.addCookie(cookie);
				}
			}

	4) 세션(Session)
		세션은 클라이언트가 서버에 요청을 보내면 자동으로 생성되며, 서버에서 유효 시간을 따로 설정하지 않았다면 브라우저를 종료할 때 자동으로 삭제된다
		서버는 클라이언트들의 상태를 각각의 세션을 통해 저장하고 클라이언트가 요청을 보낼 시 요청에 세션 ID가 같이 정달되어 동일한 ID를 가진 세션을 찾아 구분한다

	5) 동작 방식	
		- 클라이언트는 서버에 요청을 보낸다
		- 서버는 클라이언트를 식별할 수 있는 세션 id를 발급한다
		- 세션 id를 클라이언트에게 쿠키로 전송한다
		- 이 후 클라이언트는 요쳥을 보낼 때 마다 세션 id를 담은 쿠키를 보내게 된다
		- 서버에서는 세션 id를 서버에 있는 세션과 비교하여 사용자를 식별한다
		- 그 외에도 세션에 데이터를 저장하여 사용할 수 있다

	6) 세션의 사용
		- 세션 얻어오기
		HttpSession session = request.getSession();  // 현제 요청에 대한 세션을 얻는다

		- 세션에 데이터 저장하기
		session.setAttribute("name", "홍길동");

		- 세션에 저장된 데이터를 가져오기
		String name = (String) session.getAttribute("name");  // 반환타입 : Object

		- 세션에 저장된 데이터 지우기
		session.removeAttribute("name");

		- 세션 종료하기
		session.invalidate();

		-- 로그아웃 컨트롤러
			HttpSession session = request.getSession();
			session.invalidate();
