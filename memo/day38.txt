MVC -> 쿠키와 세션

1. MVC
	Model : 데이터, 비즈니스 로직 처리
		데이터베이스와 상호작용하거나 어플리케이션 상태 등을 관리함

	View : 사용자 인터페이스를 보여주고 사용자 입력을수집함

	Controller : 사용자 입력을 받아 처리하고 적절한 Model과 View를 연결함
			비즈니스 로직을 수행하거나 Model과 View 사이의 흐름을 제어함

	모델1
		JSP 중심으로 웹 어플리케이션을 개발
		모든 로직(비즈니스 로직, 데이터 처리, 화면처리)이 JSP 파일에 포함됨
		초창기 웹 어플리케이션에서 많이 사용되던 방식

		구조	JSP가 컨트롤러와 뷰의 역할을 동시에 수행
			클라이언트 요청 -> JSP가 요청 처리하고 결과를 반환

	모델2
		MVC 페턴을 도입한 구조
		컨트롤러(주로 Servlet)와 JSP를 분리하여 역할을 나눔

		구조	클라이언트 요청 ->컨트롤러(Servlet) -> 비즈니스 로직(Model) 수행 -> View(JSP) 반환
